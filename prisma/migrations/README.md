# Vehicle Services Platform Migrations README

This directory contains the migration files for the Vehicle Services Platform database schema. Each migration file corresponds to a specific change in the database schema and is generated by Prisma Migrate.

## Migration Files

- Each migration file is timestamped and named according to the changes it implements.
- The migration files are responsible for creating, altering, or dropping tables and fields in the database.

## Running Migrations

To apply the migrations to your database, use the following command:

```
npx prisma migrate deploy
```

This command will execute all pending migrations in the order they were created.

## Rollback Migrations

If you need to rollback the last migration, you can use:

```
npx prisma migrate rollback
```

## Important Notes

- Always ensure that you have a backup of your database before running migrations, especially in production environments.
- Review the generated SQL in the migration files to understand the changes being made to the database schema.

For more information on Prisma Migrate, refer to the [Prisma documentation](https://www.prisma.io/docs/concepts/components/prisma-migrate).
